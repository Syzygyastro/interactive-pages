Connection Application Data

What a developer must submit when applying for a grid connection in Great Britain — split by network level (transmission vs distribution) and project type (generation, demand/load, storage, or mixed/co-located). Citation links open the relevant Ofgem / NESO / ENA source document.

Transmission · Generation Large generation connecting directly to the GB transmission system. Routed via NESO (the system operator) under the Connection and Use of System Code (CUSC), with technical data set by the Grid Code. Post-reform, the queue position depends on passing the Gate 2 Criteria Methodology — a filter that requires evidence of land control and planning maturity.

Transmission · Demand Large loads connecting directly to the transmission system — most prominently data centres, plus large industrial and electrification projects. Demand sits outside the Gate 2 generation/storage filter today; reform is being designed under Ofgem's Demand Connections Reform programme (Feb 2026 Call for Input). Post-Aug 2024, new directly-connected transmission demand applications receive a Transitional Offer.

  • Reform plan: Phase 1 implements a data-centre-specific financial mechanism, updates securities, and tightens readiness requirements for data centres. Phase 2 extends to other demand types in 2026/27.
Transmission · Storage Battery and long-duration storage projects connecting at transmission voltage. Treated as generation for technical purposes under the Grid Code (it can export power), but the application carries extra fields to describe the bidirectional flow and the energy-storage characteristics. Subject to the same Gate 2 readiness filter as generation.

Transmission · Mixed / Co-located Sites that combine two or more technologies — typically solar + battery, wind + battery, or generation alongside on-site demand. NESO's Guidance Notes for Co-location of Different Technologies (Issue 3.1, June 2025) set the compliance pathway. The application bundle is a stack of the per-technology forms above, plus a site-level overlay describing how the technologies share the connection.

Notes

  • The most common configuration in current applications is Consolidated, Independently Controlled — solar + BESS sharing a Grid Supply Point with each technology managing itself.
  • Adding storage behind an existing generation connection is treated as a substantial modification, not a new application — but still requires re-running compliance tests for the affected unit.

Distribution · Generation Smaller generation (broadly under ~50 MW) connecting via a Distribution Network Operator (DNO). Governed by the Distribution Code (DCode), Engineering Recommendation G99 for generation (G98 for the smallest microgenerators), and DCUSA for the commercial side. Pathway depends on the project's G99 Type: A is the smallest (≤1 MW), B is 1–10 MW, C is 10–50 MW, D is ≥50 MW or transmission-connected.

Notes

  • The G98 / G99 split is a size cut-off: G98 covers fit-and-inform installations up to 16 A per phase (typical domestic rooftop solar). G99 covers everything larger.
  • Type A applications are deliberately streamlined — the network impact of a single small generator is local. Larger Types (B / C / D) carry progressively more documentation and stage-gated commissioning.

Governed by the Distribution Code DPC sections and the DNO's own Connection Charging Methodology. Embedded demand sits outside the Gate 2 Criteria Methodology today. Most DNO storage applications follow the standard G99 Type pathway, with the additions below.
